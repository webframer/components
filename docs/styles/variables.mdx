# CSS Variables

### Spacing

`--padding` and `--margin` use relative `em` unit so that the entire UI scales dynamically according to the font size.

This is the desired and expected behavior for most use cases, because it eliminates the bloated code required to style all UI Components for different sizes.

For example, in order to scale the [Switch](/docs/ui/inputs/Switch) input, all we have to do is set the font size:

<CodeExample>
  <View className='gap'>
    <Switch className='font-smallest' value={true} />
    <Switch className='font-smaller' value={true} />
    <Switch className='font-small' value={true} />
    <Switch value={true} />
    <Switch className='font-large' value={true} />
    <Switch className='font-larger' value={true} />
    <Switch className='font-largest' value={true} />
  </View>
</CodeExample>
